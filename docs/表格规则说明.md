# 表格规则说明

这份文档说明 `table_rules.csv` 中可用的表格规则语法。

## 表头结构

`table_rules.csv` 固定使用 6 列：

```csv
rule_id,priority,match_type,match_value,target_property,target_value
```

字段含义：

- `rule_id`：规则唯一标识
- `priority`：优先级，数值越小越先执行
- `match_type`：当前常用 `regex`
- `match_value`：当前常用 `.*`
- `target_property`：表格定位方式和目标属性
- `target_value`：目标值

## 当前支持的目标属性

表格规则当前支持这些属性：

- `font_name`
- `font_size`
- `bold`
- `alignment`
- `vertical_alignment`
- `border`

其中：

- `font_name`：字体名
- `font_size`：字号，例如 `10.5pt`
- `bold`：是否加粗，取值 `true / false`
- `alignment`：段落水平对齐，取值 `left / center / right / justify`
- `vertical_alignment`：单元格垂直对齐，取值 `top / center / bottom / both`
- `border`：单元格边框样式，当前支持 `single / none`

这些属性都可以和不同的定位语法组合使用。

## 1. 整表规则

作用于整张表：

```csv
TBL-001,10,regex,.*,font_name,SimSun
TBL-002,11,regex,.*,font_size,10.5pt
TBL-003,12,regex,.*,bold,true
TBL-004,13,regex,.*,alignment,center
TBL-005,14,regex,.*,vertical_alignment,center
TBL-006,15,regex,.*,border,single
```

## 2. 表头行 / 数据行

```csv
TBL-007,16,regex,.*,header_row_font_name,SimHei
TBL-008,17,regex,.*,header_row_bold,true
TBL-009,18,regex,.*,header_row_vertical_alignment,center
TBL-010,19,regex,.*,body_rows_font_size,10.5pt
TBL-011,20,regex,.*,body_rows_alignment,center
TBL-012,21,regex,.*,body_rows_border,none
```

含义：

- `header_row_*`：只作用第 1 行
- `body_rows_*`：只作用第 2 行到最后一行

## 3. 指定列

```csv
TBL-013,22,regex,.*,column[0]_font_name,SimHei
TBL-014,23,regex,.*,column[0]_bold,true
TBL-015,24,regex,.*,column[0]_vertical_alignment,center
```

含义：

- `column[0]` 是第 1 列
- `column[1]` 是第 2 列
- 索引从 `0` 开始

## 4. 指定单元格

```csv
TBL-016,25,regex,.*,cell[1,1]_font_size,14pt
TBL-017,26,regex,.*,cell[0,1]_alignment,center
TBL-018,27,regex,.*,cell[0,0]_border,single
```

含义：

- `cell[row,col]`
- `cell[1,1]` 表示第 2 行第 2 列

## 5. 按表头文本定位列

```csv
TBL-019,28,regex,.*,column_by_header[项目二]_font_name,SimHei
TBL-020,29,regex,.*,column_by_header[项目二]_alignment,center
```

含义：

- 只检查第一行
- 找到文本精确等于 `项目二` 的表头单元格
- 然后对这一整列应用规则

边界：

- 只认第一行
- 只认精确文本
- 不做模糊匹配
- 找不到时会在报告中标记 `unresolved`

## 6. 指定列范围

```csv
TBL-021,30,regex,.*,column_range[0:2]_font_size,14pt
TBL-022,31,regex,.*,column_range[0:2]_vertical_alignment,center
```

语义：

- 左闭右开
- 命中 `0 <= 列索引 < 2`

也就是命中：

- 第 1 列
- 第 2 列

不包含第 3 列。

## 7. 指定行

```csv
TBL-023,32,regex,.*,row[1]_font_name,SimHei
TBL-024,33,regex,.*,row[1]_bold,true
TBL-025,34,regex,.*,row[1]_border,none
```

含义：

- `row[0]` 是第 1 行
- `row[1]` 是第 2 行

## 8. 指定行范围

```csv
TBL-026,35,regex,.*,row_range[0:2]_font_size,14pt
```

语义同样是左闭右开：

- 命中第 1 行和第 2 行
- 不包含第 3 行

## 9. 指定矩形区域

```csv
TBL-027,36,regex,.*,cell_range[0:2,0:2]_font_name,SimHei
TBL-028,37,regex,.*,cell_range[0:2,0:2]_alignment,center
TBL-029,38,regex,.*,cell_range[0:2,0:2]_border,single
```

语义：

- 行范围：`0:2`
- 列范围：`0:2`
- 两个维度都采用左闭右开

表示命中这个矩形区域：

- 第 1 行到第 2 行
- 第 1 列到第 2 列

## 索引与区间规则

整套表格 DSL 统一遵循两条规则：

- 行列索引从 `0` 开始
- 所有 `range[start:end]` 都是左闭右开

例如：

- `row_range[0:1]` 只命中第 1 行
- `column_range[1:3]` 命中第 2 列和第 3 列
- `cell_range[0:2,1:3]` 命中前两行、后两列组成的矩形

## 失败行为

如果发生这些情况：

- 索引越界
- 范围越界
- `column_by_header[...]` 找不到对应表头

系统会：

- 不偷偷改别的单元格
- 不使用启发式猜测
- 在修改报告里显式标记 `unresolved`

## 推荐写法

推荐从粗到细叠加规则：

1. 先写整表默认规则
2. 再写表头行 / 数据行规则
3. 最后写列、行、单元格、矩形区域这类特殊规则

例如：

```csv
rule_id,priority,match_type,match_value,target_property,target_value
TBL-001,10,regex,.*,font_name,SimSun
TBL-002,11,regex,.*,font_size,10.5pt
TBL-003,12,regex,.*,header_row_bold,true
TBL-004,13,regex,.*,header_row_vertical_alignment,center
TBL-005,14,regex,.*,body_rows_alignment,center
TBL-006,15,regex,.*,column_by_header[姓名]_font_name,SimHei
TBL-007,16,regex,.*,row[1]_border,none
TBL-008,17,regex,.*,cell_range[1:3,0:2]_alignment,center
```

这种写法既有整表默认值，也有局部显式覆盖，而且所有覆盖都是可审计的。
